# Использование input.json

## Описание

Файл `input.json` заменяет все интерактивные запросы `input()` на чтение значений из конфигурационного файла. Это позволяет:

- ✅ Автоматизировать выполнение программы
- ✅ Использовать разные конфигурации для разных сценариев
- ✅ Версионировать настройки через Git
- ✅ Легко изменять параметры без редактирования кода

## Структура файла

Файл `input.json` содержит следующие разделы:

### 1. `suzun` - Параметры для блока Сузун
- `G_payaha` - значение G_пайяха
- `G_suzun_tng` - значение G_сузун_тнг
- `K_g_suzun` - коэффициент K_g_сузун
- `manual_V_upn_suzun` - ручное значение (null = использовать расчетное)
- `manual_V_suzun_vslu` - ручное значение (null = использовать расчетное)

### 2. `lodochny` - Параметры для блока Лодочный
- `G_ichem` - значение G_ичем
- `K_otkachki` - коэффициент откачки
- `K_gupn_lodochny` - коэффициент K_G_УПН_Лодочный
- `K_g_tagul` - коэффициент K_g_tagul
- `manual_V_upn_lodochny` - ручное значение (null = использовать расчетное)
- `manual_G_sikn_tagul` - ручное значение (null = использовать расчетное)
- `manual_V_tagul` - ручное значение (null = использовать расчетное)

### 3. `cppn_1` - Параметры для блока ЦППН-1
- `manual_V_upsv_yu` - ручное значение (null = использовать расчетное)
- `manual_V_upsv_s` - ручное значение (null = использовать расчетное)
- `manual_V_upsv_cps` - ручное значение (null = использовать расчетное)

### 4. `rn_vankor` - Параметры для блока РН-Ванкор
- Все параметры `manual_F_bp_*` - ручные значения (null = использовать расчетные)

### 5. `sikn_1208` - Параметры для блока СИКН-1208
- `K_delte_g_sikn` - коэффициент К_G_sikn

### 6. `tstn` - Параметры для блока ТСТН
- Все коэффициенты `K_*` и значения `F_*`, `G_*`

### 7. `validation` - Параметры валидации
- `auto_accept_validation` - автоматически принимать значения при валидации
- `auto_replace_K_otkachki` - автоматически заменять K_откачки
- `delivery_period_e_*` - периодичность сдачи (для разных блоков)

### 8. `manual_corrections` - Ручные коррекции
- Значения для исправления ошибок валидации

## Использование

### Базовое использование

1. Отредактируйте файл `input.json` и укажите нужные значения
2. Запустите программу как обычно - все значения будут взяты из файла

```bash
python main.py
```

### Значения по умолчанию

- Если файл `input.json` отсутствует, будут использованы значения по умолчанию (0.0 для числовых, null для опциональных)
- Если параметр отсутствует в файле, будет использовано значение по умолчанию

### Значение `null`

Параметры со значением `null` означают:
- Для `manual_*` параметров: использовать расчетное значение
- Для других параметров: использовать значение по умолчанию (обычно 0.0)

## Примеры

### Пример 1: Установка конкретных значений

```json
{
  "suzun": {
    "G_payaha": 100.5,
    "G_suzun_tng": 200.0,
    "K_g_suzun": 0.05
  }
}
```

### Пример 2: Использование расчетных значений

```json
{
  "suzun": {
    "manual_V_upn_suzun": null,
    "manual_V_suzun_vslu": null
  }
}
```

### Пример 3: Ручная коррекция

```json
{
  "suzun": {
    "manual_V_upn_suzun": 5000.0
  }
}
```

## Миграция со старого кода

Старый код с `input()`:
```python
G_payaha = float(input("Введите значение G_пайяха: "))
```

Новый код автоматически читает из `input.json`:
```python
from inputs import get_suzun_inputs
inputs = get_suzun_inputs()
G_payaha = inputs["G_payaha"]
```

## Примечания

- Файл `input.json` должен быть в кодировке UTF-8
- Все числовые значения должны быть числами (не строками)
- `null` в JSON соответствует `None` в Python
- При изменении файла перезапустите программу
